your explanation

TPR = TP / (TP+FN),
FPR = FP / (FP+TN)

precision = TP / (TP+FP),
recall = TP / (TP+FN)

Assume we have a imbalanced dataset, where positive class is very rare compared to negative class. One of the real examples is anomaly activities in data center. Say there is 100 incidents in 2 million recorded data.

And we have 2 classifying algorithm:

algo 1: identified 90 over 100
algo 2: identified 90 over 1000

in ROC,

algo 1: TPR = 90/100=0.9, FPR = 10/1999900
algo 2: TPR = 90/100=0.9, FPR = 910/1999900
The difference in FPR is negligible

in PR,

algo 1: precision = 90/100=0.9, recall = 0.9
algo 2: precision = 90/1000=0.09, recall = 0.9
The difference in precision is 0.81, which is much obvious than ROC

So the PR has a better result than ROC in imbalanced dataset, especially those with rare positive cases

from scipy.misc import comb
import math
import numpy as np

def ensemble_error(num_classifier, base_error):
    k_start = math.ceil(num_classifier/2)
    probs = [comb(num_classifier, k)*(base_error**k)*((1-base_error)**(num_classifier-k)) for k in range(k_start, num_classifier+1)]
    return sum(probs)

def plot_base_error(ensemble_error_func, num_classifier, error_delta):

    error_range = np.arange(0.0, 1+error_delta, error_delta)
    ensemble_errors = [ensemble_error_func(num_classifier=num_classifier, base_error=error) for error in error_range]

    plt.plot(error_range, ensemble_errors, 
             label = 'ensemble error',
             linewidth=2)
    plt.plot(error_range, error_range,
             label = 'base error',
             linestyle = '--',
             linewidth=2)
    plt.xlabel('base error')
    plt.ylabel('base/ensemble error')
    plt.legend(loc='best')
    plt.grid()
    plt.show()
